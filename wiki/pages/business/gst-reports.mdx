# GST Reports

The GST Reports module generates **GSTR-1 and GSTR-3B style** GST reports from the invoices stored in Firestore. These reports help accountants file monthly/quarterly GST returns.

**File:** `src/pages/GSTReportsPage.tsx` (19KB)

import { Callout } from 'nextra/components'

<Callout type="info">
KaranArjun does not directly file GST returns — it generates the computed data that an accountant can use to manually file on the GST portal.
</Callout>

## Report Filters

- **Period**: Month + Year picker, or custom date range
- **Supply Type**: B2B only / B2C only / All
- **State**: Filter intra-state vs inter-state supplies
- **Status**: Include only Paid invoices / All invoices

## GST Slab Breakdown

The report groups invoices by GST slab:

```
5%  slab:  ₹2,00,000 taxable → ₹10,000 GST
12% slab:  ₹1,50,000 taxable → ₹18,000 GST
18% slab:  ₹1,95,000 taxable → ₹35,100 GST
28% slab:  ₹0 taxable        → ₹0 GST
```

## How It Works

```typescript
// GSTReportsPage.tsx — aggregation logic
const gstSummary = invoices.reduce((acc, inv) => {
  inv.items.forEach(item => {
    const key = `gst_${item.gstRate}`;
    acc[key] = acc[key] || { taxable: 0, cgst: 0, sgst: 0, igst: 0 };
    acc[key].taxable += item.taxableValue;
    acc[key].cgst    += item.cgst;
    acc[key].sgst    += item.sgst;
    acc[key].igst    += item.igst;
  });
  return acc;
}, {});
```

No server-side aggregation — all computation happens client-side from the Firestore query results.
