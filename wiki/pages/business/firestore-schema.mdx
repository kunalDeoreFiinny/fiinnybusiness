# Firestore Schema

KaranArjun uses Firebase Firestore as its primary database. All data is scoped under `tenants/{tenantId}/` to ensure strict multi-tenant isolation.

## Top-Level Collections

```
Firestore
├── users/               ← All user accounts (cross-tenant)
├── tenants/             ← One doc per business
├── aiUsage/             ← AI rate limiting (cross-tenant)
└── paymentLinks_public/ ← Public payment page data
```

## Firestore Security Rules

All authenticated reads/writes require:
1. `request.auth != null` — user is logged in
2. Tenant scoping — user's `tenantId` must match the path's `tenantId`

Example rule:
```javascript
match /tenants/{tenantId}/{document=**} {
  allow read, write: if request.auth != null &&
    get(/databases/$(database)/documents/users/$(request.auth.uid))
      .data.tenantId == tenantId;
}
```
